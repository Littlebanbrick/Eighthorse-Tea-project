# 系统架构

### **一、四层功能架构**

系统按递进逻辑分四层，每一层都要能独立触发、独立输出结果，对应不同使用场景：

1. **成分 → 感知**：科学映射关系。场景：向客户解释成分和风味的具体关联。（对应成分党中建立“成分→感知”体系的需求。）

2. **感知 → 具象化语言**：把"好喝""香"这种模糊感知，翻译成具体、可传播的国内描述语言。场景：门店/国内用户讲解风味。（对应真实挑战中滋味难表达的需求）

3. **具象化语言 → 跨文化表达**：面向欧美等海外受众，做分受众的类比翻译（对照咖啡/红酒体系）。场景：海外线下面对面推销的基础物料。（对应出海难翻译的需求，适配营销党中“针对不同文化预警，设计传播话术”的需求）\(海外线上线下均可用）

4. **跨文化表达 → 多模态物料**：把跨文化语言做成图片、视频等实际可传播的内容。场景：TikTok等社媒自媒体运营。（对应营销党中“生成多语言多模态的物料”的需求）

### **二、需要补充完善的四个点**

1. **技术架构要做成"四个独立模块"，不是一条写死的流水线**——业务人员可以只调用某一层，不需要每次都跑完整流程。这个要提前跟技术同学对齐，是架构设计的前提。

> 这个点我当时是和ai这么表述的：
> 我希望这四层功能都是能够层层追溯、每一层都能独立输出结果的。  因为在不同的场景下，我需要的东西不一样： 
> 1\. 第一层成果（成分与关联）：比如我要向客户解释，这个茶里面的成分和风味到底有什么具体的关联，那我只需要第一层的成果。
> 2\. 第二层成果（风味描述）：如果我只是需要向国内的用户解释，要用什么样的语言去描述这些茶的风味，那就只需要第二层的成果。 
> 3\. 第三层成果（基础物料）：如果我需要向国外的用户推销（比如面对面推销），需要一些基础的底层物料或资料，那就需要第三层的成果。 
> 4\. 第四层成果（营销宣传）：如果我是做公司 TikTok 上面的自媒体运营，需要用这些物料去做一些营销和宣传，那我就需要用到第四层的成果。

4、 最终技术架构可以这样表述：

本项目的数据底座由两部分组成：
